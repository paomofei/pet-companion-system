# 伴读宠物系统 - 产品架构设计（V4 极简版）

> **V4 核心简化**：去掉 pending_draws 计数器。XP 满格→升级（余数保留）→🎁亮起→点击即抽。抽奖不经过中间界面。导航横幅改为许愿管理入口。

---

## 1. 系统架构总览

**分层说明**：
- **客户端层**：React SPA，Zustand 全局状态，IndexedDB 离线缓存（抽奖仅在线；首次安装生成并本地持久化 `X-Device-Id` UUID）
- **API 网关**：鉴权（V1 设备号静默注册；首次 `/users/init` 后绑定 `users.device_id`）、参数校验、响应格式化
- **服务端层**：7 个业务模块（用户/任务/目标/宠物/资产/许愿池/成就）
- **数据层**：PostgreSQL + Cron 定时任务

---

## 2. 核心业务闭环（V4 极简）

**V4 核心闭环**：`设定目标 → 任务完成 → 获得能量⚡ → 陪伴宠物 → XP满格升级（余数保留） → 🎁亮起 → 点击即抽`

> **关键简化**：
> - 无抽奖券/代币，无 pending_draws 计数器
> - pets 表上 `pending_draw BOOLEAN`：升级置 true，抽奖后置 false
> - XP 满格升级后保留余数
> - 🎁点击 = 直接抽奖（无中间界面）
> - 导航横幅 = 许愿管理（查看/新增/编辑/删除心愿），不触发抽奖

| 环节 | 触发 | 数据变更 |
|------|------|---------|
| ① 任务打卡 | 勾选复选框 | `tasks.status=1` + `users.energy_balance += N` + 流水 |
| ② 宠物互动 | 点击道具 | `users.energy_balance -= N` + `pets.current_xp += M` |
| ③ **XP满格升级** | XP >= max_xp | `pets.level += 1` + `pets.current_xp = 余数` + `pets.pending_draw = true` + 🎁亮起 |
| ④ **点击🎁** | 用户主动 | 在线时生成 `clientRequestId` 并请求抽奖，处理中播放翻牌动效 |
| ⑤ **抽奖完成** | 系统自动 | **加权随机**命中 → `wishes.status=1` + `pets.pending_draw = false` + 🎁变灰 |

## 3. 页面路由与模块映射

| 路由 | 模块 | 核心功能 |
|------|------|---------|
| `/onboarding` | 引导页 | 昵称输入 + 开局选择 + 宠物领养 |
| `/today?tab=tasks` | 任务页 (默认) | 日期导航 + 任务 CRUD + 按周指定重复 + 打卡 Aha Moment |
| `/today?tab=goals` | 目标页 | 目标 CRUD + 进度条分色 |
| `/today?tab=growth` | 成长页 | 高光数据 + 7日柱状图 + 勋章墙 |
| 常驻左侧 | **宠物面板** | 资产栏(仅⚡) + 宠物立绘 + 气泡对白 + **XP条+🎁礼物盒** + 道具互动 |
| 🎁点击触发 | **抽奖动效** | 点击🎁 → 直接翻牌抽奖（无中间界面） |
| 导航横幅触发 | **许愿管理 Overlay** | 查看/新增/编辑/删除心愿 + 中奖记录 |

> **两个入口，各司其职**：
> ① XP 进度条🎁礼物盒 = **抽奖入口**（升级后亮起，点击即抽）
> ② 导航栏许愿池横幅 = **许愿管理入口**（常驻可见，管理心愿单）

## 4. XP 进度条 + 礼物盒 UX 设计

```text
 LV.3  [██████████████████████░░░░]  78/100 XP    [🎁]
                                                    ↑ 灰色=无待抽
                                                    ↑ 金色亮起=可抽

升级瞬间：进度条白光→XP重置为余数→等级放大→纸屑→🎁从灰变金+弹跳
用户点击🎁 → 在线校验 → 请求 `/wishes/draw(clientRequestId)` → 请求中播放翻牌动效 → 揭晓+庆祝 → 🎁变回灰色
```

**礼物盒二态**：

| 状态 | 视觉 | 可交互 |
|------|------|--------|
| 无待抽 (`pending_draw=false`) | 灰色🎁 | 点击提示"多陪伴宠物升级就能抽奖哦" |
| 可抽 (`pending_draw=true`) | 金色🎁 + 光晕脉冲 | 点击直接触发抽奖动效 |

## 5. 数据库 ER 模型（V4）

**12 张核心表**（V4 联调补充后）：

| 表名 | V4 变更 |
|------|---------|
| `users` | **新增** `device_id UNIQUE`；**移除** `pending_draws` 字段；资产栏仅展示⚡ |
| `pets` | **新增** `pending_draw BOOLEAN DEFAULT false` |
| `wish_draw_requests` | **新增** 抽奖幂等表，按 `user_id + client_request_id` 保证重复请求返回第一次结果 |
| `tasks` | 任务实例继续存 `target_date`；重复规则仍由模板派生 |
| `task_templates` | **新增** `start_date`、`repeat_weekdays`，支持“按周指定”重复任务 |
| `wishes` | 不变（weight + status） |
| 其余表 | 不变 |

## 7. V4 修订摘要

| 变更 | 说明 |
|------|------|
| `device_id` 静默绑定 | 首次安装生成 `X-Device-Id` UUID；首次 `/users/init` 后绑定 `users.device_id` |
| 重复任务扩展 | `repeatType` 新增“按周指定”，模板持久化 `start_date + repeat_weekdays` |
| 去掉 `pending_draws` 计数器 | users 表移除此字段，pets 表新增 `pending_draw` 布尔值 |
| XP 保留余数 | 升级后 `current_xp = (current_xp + gain_xp) % max_xp` |
| 🎁点击即抽 | 无中间 Modal，直接播放抽奖动效 |
| 导航横幅 = 许愿管理 | 不再是抽奖入口，仅用于管理心愿 |
| 横幅文案固定 | `🎁 许愿池：管理你的心愿单 [去看看 >]` |
