## 3. 核心交互要点

### 3.1 XP 进度条 + 🎁礼物盒（V4 核心简化）

```text
XP 进度条布局：
  LV.3  [██████████████████░░░░░░░]  78/100 XP    [🎁]
                                                     ↑ 礼物盒组件

礼物盒二态（V4 简化）：
├─ 灰色(pending_draw=false)：静态灰色图标，点击Toast"多陪伴宠物升级就能抽奖哦"
└─ 金色(pending_draw=true)：金色+光晕脉冲CSS动画，点击→直接触发抽奖动效

升级时序：
  T=0     互动事务返回 { leveledUp:true, pendingDraw:true }
  T=0-200 进度条白光扫过 → XP 重置为余数
  T=200   等级数字放大1.2x变色
  T=400   纸屑爆发
  T=600   🎁从灰变金 + scale(0→1.3→1.0) 弹跳
  T=800   宠物气泡P0: "升级啦！点击礼物盒看看有什么惊喜🎁"
```

### 3.2 🎁点击即抽（V4 极简）

**点击金色🎁** → 直接播放抽奖动效（无中间 Modal）。

```text
抽奖流程：
  点击🎁 → 校验奖池是否为空
  → 若空：弹出“许愿池空了”提示卡（“+许个新愿望”按钮）
  → 非空：翻牌动效(1-2s) → POST /wishes/draw
  → 返回命中心愿 → 心愿卡放大+Confetti
  → 宠物P1气泡:"哇！[心愿]中啦！"
  → pendingDraw=false → 🎁变回灰色
  → 结果展示卡（"恭喜中奖！"+"再许个新愿望"+"好的"）
```

### 3.3 导航栏许愿管理横幅

**固定文案**（不再显示抽奖次数）：
```text
  [ 📅 今日任务 ](Active)    [ 🎯 大目标 ]    [ 📈 成长轨迹 ]
  +-------------------------------------------------------------+
  | 🎁 许愿池：管理你的心愿单                    [去看看 >]    |
  +-------------------------------------------------------------+
```

点击"去看看 > "→ 打开许愿管理 Overlay（仅管理心愿，不触发抽奖）。

### 3.4 打卡飞行动效

不变：`getBoundingClientRect()` + CSS `cubic-bezier()` + Framer Motion。

### 3.5 左右面板同步（< 100ms）

Zustand 订阅，乐观更新。
