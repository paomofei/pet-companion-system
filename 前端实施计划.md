# 伴读宠物系统 - 前端实施计划（V4 极简版）

> **V4 极简**：去掉 `pending_draws` 计数器。`pets.pending_draw BOOLEAN`：升级置 true，抽奖后置 false。XP 保留余数。礼物盒二态。🎁点击即抽（无中间界面）。导航横幅 = 许愿管理入口。

## 3. 核心交互要点

### 3.1 XP 进度条 + 🎁礼物盒（V4 核心简化）

```text
XP 进度条布局：
  LV.3  [██████████████████░░░░░░░]  78/100 XP    [🎁]
                                                     ↑ 礼物盒组件

礼物盒二态（V4 简化）：
├─ 灰色(pending_draw=false)：静态灰色图标，点击Toast"多陪伴宠物升级就能抽奖哦"
└─ 金色(pending_draw=true)：金色+光晕脉冲CSS动画，点击→直接触发抽奖动效

升级时序：
  T=0     互动事务返回 { leveledUp:true, pendingDraw:true }
  T=0-200 进度条白光扫过 → XP 重置为余数
  T=200   等级数字放大1.2x变色
  T=400   纸屑爆发
  T=600   🎁从灰变金 + scale(0→1.3→1.0) 弹跳
  T=800   宠物气泡P0: "升级啦！点击礼物盒看看有什么惊喜🎁"
```

### 3.2 🎁点击即抽（V4 极简）

**点击金色🎁** → 在线时立即请求抽奖接口，请求处理中播放抽奖动效（无中间 Modal）。

```text
抽奖流程：
  点击🎁 → 校验网络
  → 若离线：Toast"网络不可用，恢复后再抽奖"
  → 若在线：生成 clientRequestId → POST /wishes/draw
  → 若空：弹出“许愿池空了”提示卡（“+许个新愿望”按钮）
  → 非空：请求处理中播放翻牌动效(1-2s)
  → 返回命中心愿 → 心愿卡放大+Confetti
  → 宠物P1气泡:"哇！[心愿]中啦！"
  → pendingDraw=false → 🎁变回灰色
  → 结果展示卡（"恭喜中奖！"+"再许个新愿望"+"好的"）
  → 若请求超时：使用同一个 clientRequestId 自动重试 1 次
```

### 3.3 导航栏许愿管理横幅

**固定文案**（不再显示抽奖次数）：
```text
  [ 📅 今日任务 ](Active)    [ 🎯 大目标 ]    [ 📈 成长轨迹 ]
  +-------------------------------------------------------------+
  | 🎁 许愿池：管理你的心愿单                    [去看看 >]    |
  +-------------------------------------------------------------+
```

点击"去看看 > "→ 打开许愿管理 Overlay（仅管理心愿，不触发抽奖）。

### 3.6 Web 布局与视觉基线

- 桌面 Web 优先：默认按 `1440 × 900` 设计，最小兼容 `1280 × 800`。
- 全局采用 `12` 栏为主的布局系统；左侧宠物面板固定/sticky，右侧内容面板独立滚动。
- 页面背景使用柔和渐变叠加极弱噪点纹理；卡片采用轻拟态圆角、轻阴影、轻高光。
- 标签切换、弹窗、空状态切换采用轻过渡动画，不使用重型花哨动效。
- 所有空状态必须带插画/图形占位和明确引导按钮。
- 所有表单统一实时内联校验，错误在输入过程中即时提示。
- 视觉关键词：`柔和 / 温暖 / 细腻 / 高级 / 童趣但不过度幼稚`。
- 过渡范围：页签切换、模块展开/收起、宠物互动反馈、弹窗出现与关闭、空状态切换到内容状态。
- 桌面验收口径：在 `1280 × 800` 与 `1440 × 900` 下，主内容无需拖动画布找内容；超长内容仅允许右侧面板内部滚动。

### 3.7 Web UI 规范

- 栅格与间距：
  - 页面边距 `12-40`
  - 主页面优先 `12` 栏，复杂密集场景可扩展到 `24` 栏
  - 栅格间距 `8-20`
  - 元素间距 `2-20`
  - 大模块优先使用 `16/20` 级间距，卡片内部优先使用 `8/12/16`
- 字体：
  - 中文优先 `PingFang SC, MiSans, Source Han Sans SC, Microsoft YaHei, sans-serif`
  - 英文/数字优先 `SF Pro Text, Helvetica, Arial, Roboto, sans-serif`
- 文字规范：
  - 中文最小字号 `12`
  - 英文/数字最小字号 `10`
  - 主标题 `24`
  - 二级标题 `18-20`
  - 卡片标题 `14-16`
  - 正文 `14`
  - 注释/辅助文案 `12`
  - 行高 `1.2-1.8`
- 图标规范：
  - 控件图标统一 `16` 或 `20`
  - 标签页图标统一 `18-20`
  - 空状态/装饰图标可放大到 `40+`
- 按钮规范：
  - 主 CTA 高度 `40-44`
  - 次按钮高度 `32`
  - 行内操作按钮高度 `24-28`
- 输入框规范：
  - 主表单输入框统一 `36-40`
  - 行内轻量输入统一 `28-32`

### 3.8 本轮设计原则

- 优先修复布局不稳，再做视觉优化。
- 优先增强陪伴感和任务聚焦度，不新增无必要选择项。
- 优先做到清晰、稳定、细腻，避免花哨和廉价感。
- 动效必须轻，不喧宾夺主。
- 儿童向表达可以温暖，但不做过度幼稚化设计。

## 4. API 接口清单（V4·26 个）

- 前端首次安装生成并本地持久化 `X-Device-Id` UUID。
