# 伴读宠物数据库设计（V4 修订版）

> **V4 修订**：
> - `users` 表新增 `device_id VARCHAR(64) NOT NULL UNIQUE`（客户端生成 UUID，首次 `/users/init` 时绑定）
> - `users.pending_draws` → **删除**（不再在 users 表上维护抽奖）
> - `pets` 表新增 `pending_draw BOOLEAN DEFAULT false`（升级后置 true，抽奖后置 false）
> - XP 满格后**保留余数**（不归零）
> - `wish_draw_requests` 表新增，用于抽奖接口幂等
> - `task_templates` 表新增 `start_date`、`repeat_weekdays`，支持“按周指定”重复规则

---

## V4 变更

### 1. users 表：新增 `device_id`，移除 `pending_draws`

```sql
CREATE TABLE users (
  id            SERIAL PRIMARY KEY,
  device_id     VARCHAR(64) NOT NULL UNIQUE,
  nickname      VARCHAR(20) NOT NULL,
  energy_balance      INT NOT NULL DEFAULT 0,
  -- V1：客户端生成的 UUID，首次 /users/init 时绑定到用户
  -- V4：移除 pending_draws（抽奖状态移至 pets 表）
  onboarding_option   SMALLINT DEFAULT 0,
  current_streak      INT NOT NULL DEFAULT 0,
  max_streak          INT NOT NULL DEFAULT 0,
  total_energy_earned INT NOT NULL DEFAULT 0,
  total_tasks_done    INT NOT NULL DEFAULT 0,
  created_at    TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at    TIMESTAMP NOT NULL DEFAULT NOW(),
  deleted_at    TIMESTAMP NULL,

  CONSTRAINT chk_energy_non_neg CHECK (energy_balance >= 0)
);
```

### 2. pets 表：新增 `pending_draw` 布尔值

```sql
CREATE TABLE pets (
  id           SERIAL PRIMARY KEY,
  user_id      INT NOT NULL UNIQUE REFERENCES users(id),
  name         VARCHAR(20) NOT NULL,
  level        INT NOT NULL DEFAULT 1,
  current_xp   INT NOT NULL DEFAULT 0,
  max_xp       INT NOT NULL DEFAULT 100,
  appearance   VARCHAR(50) NOT NULL DEFAULT 'cat_default',
  -- V4 新增：布尔值，true=有一次待抽奖,false=无
  pending_draw BOOLEAN NOT NULL DEFAULT false,
  created_at   TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at   TIMESTAMP NOT NULL DEFAULT NOW()
);
```

**Prisma 对应**：

```prisma
model User {
  deviceId String @unique @map("device_id") @db.VarChar(64)
  // V4：移除 pendingDraws 字段
}

model Pet {
  // ...
  pendingDraw Boolean @default(false) @map("pending_draw")  // V4 新增
  // ...
}
```

**V1 约束**：
- 不新增 `user_devices` 绑定表。
- 一个 `device_id` 仅绑定一个用户。
- V1 不支持换机恢复或多设备登录。

### 3. wish_draw_requests 表：抽奖幂等

```sql
CREATE TABLE wish_draw_requests (
  id                SERIAL PRIMARY KEY,
  user_id           INT NOT NULL REFERENCES users(id),
  client_request_id VARCHAR(64) NOT NULL,
  wish_id           INT NOT NULL REFERENCES wishes(id),
  pool_remaining    INT NOT NULL DEFAULT 0,
  created_at        TIMESTAMP NOT NULL DEFAULT NOW(),

  CONSTRAINT uq_wish_draw_request UNIQUE (user_id, client_request_id)
);
```

**用途**：
- 同一用户对 `POST /wishes/draw` 使用相同 `clientRequestId` 重复提交时，返回第一次抽奖结果。
- 支撑前端超时后使用同一个 `clientRequestId` 自动重试 1 次。

**Prisma 对应**：

```prisma
model WishDrawRequest {
  id              Int      @id @default(autoincrement())
  userId          Int      @map("user_id")
  clientRequestId String   @map("client_request_id") @db.VarChar(64)
  wishId          Int      @map("wish_id")
  poolRemaining   Int      @default(0) @map("pool_remaining")
  createdAt       DateTime @default(now()) @map("created_at")

  user User @relation(fields: [userId], references: [id])
  wish Wish @relation(fields: [wishId], references: [id])

  @@unique([userId, clientRequestId])
  @@map("wish_draw_requests")
}
```

### 4. task_templates 表：支持“按周指定”重复规则

```sql
ALTER TABLE task_templates
  ADD COLUMN start_date VARCHAR(10) NOT NULL DEFAULT '1970-01-01',
  ADD COLUMN repeat_weekdays SMALLINT[] NOT NULL DEFAULT '{}';
```

**字段语义**：
- `start_date`：重复任务开始生效的日期；仅从该日期起参与自动生成。
- `repeat_weekdays`：仅 `repeat_type=4` 时使用；取值 `1-7`，对应 `周一-周日`。
- 旧规则保持不变：
  - `repeat_type=1`：每天
  - `repeat_type=2`：工作日
  - `repeat_type=3`：周末
  - `repeat_type=4`：按周指定

**Prisma 对应**：

```prisma
model TaskTemplate {
  id             Int      @id @default(autoincrement())
  userId         Int      @map("user_id")
  goalId         Int?     @map("goal_id")
  title          String   @db.VarChar(50)
  rewardEnergy   Int      @default(10) @map("reward_energy")
  repeatType     Int      @map("repeat_type")
  startDate      String   @map("start_date")
  repeatWeekdays Int[]    @default([]) @map("repeat_weekdays")
  isActive       Boolean  @default(true) @map("is_active")
  createdAt      DateTime @default(now()) @map("created_at")
  updatedAt      DateTime @updatedAt @map("updated_at")
}
```

**调度规则**：
- 调度器先判断 `target_date >= start_date`。
- 再按 `repeat_type + repeat_weekdays` 判断当天是否命中生成规则。
- `repeat_type=4` 时，仅在当天星期包含于 `repeat_weekdays` 时生成实例。

## V4 升级逻辑变更

```text
旧（V3）：XP >= maxXp → level += floor(...) → XP = XP % maxXp → pending_draws += N
新（V4）：XP >= maxXp → level += 1 → XP = (XP + gainXp) % maxXp（保留余数）→ pending_draw = true
```

## 其余不变

除 `users`、`pets`、`wish_draw_requests` 外，V3 修订版中的以下所有表和逻辑在 V4 中 **完全保持不变**：

- tasks 表（target_date + status 三态 + is_delayed_copy）
- goals 表
- task_templates 表（除本轮新增 `start_date`、`repeat_weekdays` 外）
- energy_logs 表
- items 表
- interaction_logs 表
- wishes 表（weight 权重 + drawn_at）
- badges + user_badges 表
- 所有索引
- 软删除统一 deleted_at
- 级联操作规则
- 并发控制机制

**V4 的变化影响 users 表（删一字段）、pets 表（增一字段）、wish_draw_requests 表（新增一张幂等表）和 task_templates 表（新增重复规则字段）**。
