# 伴读宠物数据库设计（V4 修订版）

> **V4 修订**：
> - `users.pending_draws` → **删除**（不再在 users 表上维护抽奖）
> - `pets` 表新增 `pending_draw BOOLEAN DEFAULT false`（升级后置 true，抽奖后置 false）
> - XP 满格后**保留余数**（不归零）

---

## V4 变更

### 1. users 表：移除 `pending_draws`

```sql
CREATE TABLE users (
  id            SERIAL PRIMARY KEY,
  nickname      VARCHAR(20) NOT NULL,
  energy_balance      INT NOT NULL DEFAULT 0,
  -- V4：移除 pending_draws（抽奖状态移至 pets 表）
  onboarding_option   SMALLINT DEFAULT 0,
  current_streak      INT NOT NULL DEFAULT 0,
  max_streak          INT NOT NULL DEFAULT 0,
  total_energy_earned INT NOT NULL DEFAULT 0,
  total_tasks_done    INT NOT NULL DEFAULT 0,
  created_at    TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at    TIMESTAMP NOT NULL DEFAULT NOW(),
  deleted_at    TIMESTAMP NULL,

  CONSTRAINT chk_energy_non_neg CHECK (energy_balance >= 0)
);
```

### 2. pets 表：新增 `pending_draw` 布尔值

```sql
CREATE TABLE pets (
  id           SERIAL PRIMARY KEY,
  user_id      INT NOT NULL UNIQUE REFERENCES users(id),
  name         VARCHAR(20) NOT NULL,
  level        INT NOT NULL DEFAULT 1,
  current_xp   INT NOT NULL DEFAULT 0,
  max_xp       INT NOT NULL DEFAULT 100,
  appearance   VARCHAR(50) NOT NULL DEFAULT 'cat_default',
  -- V4 新增：布尔值，true=有一次待抽奖,false=无
  pending_draw BOOLEAN NOT NULL DEFAULT false,
  created_at   TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at   TIMESTAMP NOT NULL DEFAULT NOW()
);
```

**Prisma 对应**：

```prisma

  // ...

  // ...

  // ...
```

## V4 升级逻辑变更

```text
旧（V3）：XP >= maxXp → level += floor(...) → XP = XP % maxXp → pending_draws += N
新（V4）：XP >= maxXp → level += 1 → XP = (XP + gainXp) % maxXp（保留余数）→ pending_draw = true
```

## 其余不变

V3 修订版中的以下所有表和逻辑在 V4 中 **完全保持不变**：

- tasks 表（target_date + status 三态 + is_delayed_copy）
- goals 表
- task_templates 表
- energy_logs 表
- items 表
- interaction_logs 表
- wishes 表（weight 权重 + drawn_at）
- badges + user_badges 表
- 所有索引
- 软删除统一 deleted_at
- 级联操作规则
- 并发控制机制

**V4 的变化影响 users 表（删一字段）和 pets 表（增一字段）**。
